# 👅 โครงร่างนวัตกรรม (Prototype) ฉบับสมบูรณ์: WebApp เกมบริหารกล้ามเนื้อใบหน้าและปาก

**ชื่อโปรเจกต์** : FaceFit
**โปรเจกต์:** ระบบกายภาพบำบัดกล้ามเนื้อที่ใช้ในการพูด (Dysarthria) สำหรับผู้ป่วย Stroke
**เทคโนโลยี:** HTML5, CSS3, Vanilla JavaScript + MediaPipe Face Mesh

## 🌟 ฟีเจอร์ในเวอร์ชันนี้

1. **Real-time Face Tracking**: ตรวจจับใบหน้าและวาดจุดริมฝีปาก (Lip Landmarks)
2. **Speech Recognition Integration**: ใช้ Web Speech API รับฟังและวิเคราะห์ความแม่นยำในการออกเสียงผู้ป่วย
3. **Action Detection**: ตรวจจับท่าทาง 2 ท่าหลัก คือ **"อ้าปากกว้าง"** และ **"ฉีกยิ้ม"** นำมาใช้คำนวณคะแนนโบนัสร่วมกับการออกเสียง
4. **Game Logic & Score**: มีระบบสุ่มคำสั่ง หากผู้ป่วยพูดชัดเจนและทำท่าทางปากถูกต้อง หลอดพลังจะเต็มและได้คะแนนบวกเพิ่ม

> **⚠️ หมายเหตุเกี่ยวกับข้อจำกัด (Known Limitations):**
> เทคโนโลยี Web Speech API อาจทำงานได้ไม่คงที่หรือไม่สมบูรณ์ในแอป Safari บนอุปกรณ์ iOS (iPhone/iPad) แนะนำให้ผู้ใช้งานรันเว็บบนเบราว์เซอร์ **Google Chrome** หรือบนอุปกรณ์ Android / PC เพื่อความแม่นยำสูงสุดในการประมวลผลเสียงพูด

## 🧮 หลักการคำนวณ (คณิตศาสตร์เบื้องหลัง)

ระบบใช้สมการหาระยะห่างระหว่างจุดสองจุด (Euclidean Distance) เพื่อประเมินรูปปากของผู้ป่วย:
$$d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$$

* **อ้าปาก (Mouth Open):** วัดระยะห่างระหว่างริมฝีปากบนด้านใน (จุด 13) และริมฝีปากล่างด้านใน (จุด 14)
* **ฉีกยิ้ม (Smile):** วัดระยะห่างระหว่างมุมปากซ้าย (จุด 61) และมุมปากขวา (จุด 291)

## 🏗️ ภาพรวมโครงสร้างระบบ (System Architecture)

### 1. Frontend & AI Processing (ส่วนหน้าบ้านและการประมวลผล)

ทำงานบนเบราว์เซอร์ของผู้ใช้ (Client-side) ทั้งหมดเพื่อไม่ให้เซิร์ฟเวอร์ทำงานหนัก:

* **User Interface:** HTML/CSS (`index.html`, `style.css`) รองรับ PWA ให้ผู้ป่วยติดตั้งลงมือถือได้
* **AI Models:** ประมวลผลผ่าน `game.js` โหลด MediaPipe Face Mesh และ Web Speech API ทำงานบนมือถือโดยตรง

### 2. Backend & Database (ระบบฐานข้อมูลและการจัดการผู้ใช้)

ใช้บริการ Backend-as-a-Service (BaaS) อย่าง **Supabase**:

* **Authentication:** ระบบเข้าสู่ระบบสำหรับผู้ป่วย (PIN/เบอร์โทร) และแพทย์/นักกายภาพ
* **PostgreSQL Database:** เก็บข้อมูล 3 ตารางหลัก คือ ข้อมูลผู้ป่วย, ชุดคำศัพท์, และประวัติการทำกายภาพ (คะแนนและสถิติ)

### 3. Workflow & Automation (ระบบจัดการอัตโนมัติ)

ใช้เครื่องมือ **n8n** เพื่อเป็นตัวกลางและทำ Automation:

* **Daily/Weekly Reports:** ดึงข้อมูลจาก Supabase สรุปผลประจำสัปดาห์
* **Notifications:** แจ้งเตือนสถิติผ่าน LINE Notify หรือ Email ให้แพทย์หรือญาติผู้ป่วย

### 4. Hosting (ที่เก็บไฟล์หน้าเว็บ)

* **Vercel / Netlify:** ฝากไฟล์ Frontend อัปเดตอัตโนมัติเมื่อแก้โค้ดจาก GitHub

## 📂 วิธีการทำงานของระบบ (System Flow)

1. ผู้ป่วยเปิดแอป FaceFit บนมือถือ ระบบ `auth.js` ตรวจสอบสถานะการล็อกอิน
2. ดึงชุดคำศัพท์ที่ตั้งค่าไว้จาก Supabase ผ่าน `db.js`
3. ผู้ป่วยเริ่มเล่นเกม `game.js` ทำงานร่วมกับกล้องและไมค์เพื่อประเมินคะแนน
4. เมื่อเล่นจบ `db.js` จะส่งข้อมูลคะแนน ท่าที่ทำสำเร็จ และช้าม กลับไปบันทึกที่ Supabase
5. ระบบ n8n ส่งสรุปผลรายวันผ่านเดตไลน์ครอบครัวหรือ LINE Notify (ส่วนออกแบบสำหรับอนาคต)
